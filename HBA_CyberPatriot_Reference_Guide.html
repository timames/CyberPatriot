üìÖ SCHEDULED TASKS - BACKDOOR CHECK CRITICAL

üö® HIGH BACKDOOR RISK!

Scheduled tasks are a common persistence mechanism. Check these CAREFULLY!

# List all scheduled tasks to file for analysis schtasks /query /v /fo list > C:\Users\Public\schtasksBackup.txt
# Find suspicious tasks (tasks with file paths in "Task To Run") findstr /i "C:\" C:\Users\Public\schtasksBackup.txt notepad C:\Users\Public\schtasksBackup.txt
# Look for tasks created by users (not Microsoft) # Search for custom authors in the backup file # Look for hostnames or usernames in "Author:" field # Microsoft tasks typically have "Author: Microsoft Corporation"
# Common signs of malicious scheduled tasks: # - Task runs at logon or frequently (every few minutes) # - Task runs as SYSTEM user # - Task runs executables from: # - C:\Users\[user]\AppData\ # - C:\Windows\Temp\ # - C:\Logs\ # - Task has suspicious names # - Task author is not Microsoft # - Task created recently # Example malicious indicators: # "Task To Run: C:\Logs\tini.exe" # "Task To Run: C:\Users\user\AppData\Roaming\bd.exe" # "Author: [ComputerName]\[username]"
# Open Task Scheduler GUI (easier to review) taskschd.msc
# Delete specific malicious task schtasks /delete /tn "TaskName" /f
# View recently modified tasks dir /o-d C:\Windows\System32\Tasks\ | more dir /o-d C:\Windows\System32\Tasks\Microsoft\Windows\ | more
# ‚ö†Ô∏è BEFORE deleting tasks, find and delete the malicious file first! # Note the file path from "Task To Run" field # Delete the malicious .exe or .bat file # THEN delete the scheduled task # Otherwise points check might fail!

üîê PASSWORD POLICY HIGH VALUE

‚ö†Ô∏è SET POLICY BEFORE CHANGING PASSWORDS!
